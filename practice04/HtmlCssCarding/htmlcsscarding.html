HTML

主流浏览器及其内核

  • IE: trident内核
  • Firefox: gecko内核
  • Safari: webkit内核
  • Opera: 以前是presto内核, Opera现已改用Google Chrome的Blink内核
  • Chrome: Blink(基于webkit, Google与Opera SoftWare共同开发)

Doctype是什么及其特点

  • <!doctype>必须处于HTML文档的头部，在标签之前
  • <!doctype>声明不是一个HTML标签，用于告诉浏览器使用哪种HTML或XHTML规范
  • 该标签可以声明三组DTD类型，分别表示阉割版本、过渡版本以及基于框架的版本

DIV + CSS的布局较TABLE布局的优势

  • 利于重构，只需要更改CSS文件
  • 页面加载速度更快、结构化清晰、页面显示简洁
  • 表现与结构分离
  • 易于优化（SEO）搜索引擎支持更友好，排名更容易靠前
  • Chrome: Blink(基于webkit，Google与Opera SoftWare共同开发)

img的title和alt区别

  • title用于为元素提供附加的提示，鼠标滑动到元素上会显示title文字
  • alt是img特有的属性，是图片内容的自定义描述，用于图片无法加载时，描述图片内容
  • 搜索引擎会分析alt描述

src与href的区别

  • src是source的缩写，指向外部资源的位置，指向的内容将会嵌入下载到文档中当前的标签位置，如js，img，frame等元素
    <script src="*.js"></script>
    当浏览器解析到该元素时，会暂停其他资源的下载和处理，知道将该资源加载、编译、执行完毕，图片和框架等元素也如此。这也是js脚本要放在底部而不是头部的原因
  • href是Hypertext Reference的缩写，指向网络资源所在的位置，建立和当前元素（锚点）或当前文档（链接）之间的链接
    <link href="*.css" rel="stylesheet/>
    当浏览器解析到该元素时，就会并行下载资源并且不对停止对当前文档的处理。这也是建议用link方式加载css，而不是使用@import方式

web语义化

  • 是指通过HTML标记表示页面包含的信息，包含了HTML标签的语义化和CSS命名的语义化
  • HTML标签的语义化: 使用例如H1-H6等恰当的表达文档结构
  • CSS命名的语义化: 为HTML标签添加有意义的class，id
  • 需要语义化的原因:
    • 去掉样式后页面依然呈现清晰的结构
    • 盲人使用读屏器更好的阅读
    • 搜索引擎更好的理解页面，有利于收录
    • 团队项目可持续运作和运维

行内元素、块级元素、空元素有哪些

  • 行内元素: display默认属性为inline，主要有 a, b, span, img, input, select, strong
  • 块级元素: display默认属性为block，主要有 div, ul, ol, li, dl, dt, dd, h1-h6, p
  • 空元素: 在开始标签关闭，没有内容，主要有 br, hr, img, input, link, meta

cookies, sessionStorage, localStorage的区别

  • cookie: 在浏览器和服务器之间来回传递，大小受限，每次请求一个新的页面都会被发送过去，不能跨域调用
  • sessionStorage: 会话级别的存储，用于本地存储session中的数据，存储的数据只有在同一回合的页面才能访问，会话结束后数据也随之销毁
  • localStorage: 持久化的本地存储，存储本地数据，存储容量大

一个页面有大量图片，如何提高图片加载速度

  • 图片懒加载，在页面上的未可是区域添加滚动条时间，判断图片位置与浏览器顶端的距离与页面的距离，如果前者小于后者，优先加载
  • 如果为幻灯片、相册等。可以使用图片预加载技术，将当前展示图片的前一张和后一张优先下载
  • 如果为css图片，可以使用CSSsprite, SVGsprite, Iconfont, Base64等技术
  • 如果图片过大，可以使用特殊编码的图片，加载时会先加载一张压缩的缩略图
  • 如果图片展示区域小于真实图片大小，则应在服务器端根据业务需要先进行图片压缩，图片压缩后与展示一致

以前端角度SEO需要考虑什么

  • 了解搜索引擎如何抓取网页和索引网页
  • meta标签优化，包括: title, description, keywords, author, category, language等
  • 关键词分析，确定网站关键词5个左右，针对关键词进行优化，包括关键词密度density 相关度relevance 突出性prominence
  • 了解主要的搜索引擎及其对页面的抓取和索引、排序的规则，还要了解个搜索门户和搜索引擎之间的关系
  • 主要的互联网目录
  • 按点击付费的搜索引擎
  • 搜索引擎登录
  • 链接交换和链接广泛度（Link Popularity）
  • 合理的标签使用

iframe的缺点

  • iframe会阻塞页面的Onload事件
  • iframe和页面共享连接池，浏览器对相同域的链接有限制，因此会影响页面并行加载
  • 解决方法: 通过javascript动态给iframe添加src属性值

html5新增的特性

  • 用于会话的canvas元素
  • 用于媒介回放的video和audio元素
  • 对本地离线存储的更好支持的localStorage, sessionStorage
  • 新的特殊内容元素: article, footer, header, nav, section
  • 新的表单控件: calendar, date, time, email, url, search

html5移除的特性

  • 纯表现的元素: basefont, big, center, font, s, strike, tt, uu
  • 对可用性产生负面影响的元素: frame, frameset, noframes

CSS


对DOM设置CSS样式的方法

  • 外部样式表，引入一个外部的css文件
  • 内部样式表，将css代码放在<head>标签内部
  • 内联样式，将css样式直接定义在HTML元素内部

CSS有哪些选择器及优先级

  • id选择器（#idname）
  • 类选择器（.classname）
  • 相邻选择器（h1 + p）
  • 后代选择器（li a）
  • 通配符选择器（*）
  • 属性选择器（a[rel="external"]，属于CSS2，IE6不支持）
  • 伪类选择器（a:hover, li:nth-child）
  • 优先级: !important > id > class > tag, important优先级高于内联样式

CSS可以通过哪些属性定义，使得一个DOM元素不显示在浏览器可视范围

  • 设置display属性为none，或者visibility属性为hidden
  • 设置宽高为0，设置透明度为0，设置z-index在-1000

超链接访问过后hover样式就不出现，原因是什么，怎么解决

  • 原因: 被点击访问过的超链接样式不再具有hover和active样式了
  • 解决: 改变CSS属性的排列顺序: L-V-H-A(link, visited, hover, active)

行内元素和块级元素的具体区别

  • 行内元素: 和相邻的内联元素在同一行，宽度、高度、内边距和外边距的上和下都不可改变，左右可以改变
  • 块级元素: 独占一行，宽度、高度、内边距、外边距都可以改变
  • 浏览器默认的inline-block元素: 有内在尺寸可设置宽高，不换行如input, img, button, textarea, label

rgba()和opacity的透明效果有什么不同

  • rgba: 只作用于元素的颜色或背景色，子元素不会继承该效果
  • opacity: 作用于元素，以及元素内的所有内容的透明度

CSS中让文字在垂直和水平方向上的重叠的两个属性是什么

  • 垂直方向: line-height
  • 水平方向: letter-spacing，可用于消除inline-block元素间的换行符空格间隙问题

如何垂直居中一个img（简便方法）

  • display: table-cell;
  • text-align: center;
  • vertical-align: middle;

px和em的区别

  • px: 固定长度单位，指定多少即为多少，易于计算
  • em: 不固定长度单位，继承父级元素的字体大小
  • 浏览器默认的字体高度为16px，所以未经调整的浏览器都符合1em=16px

display:none与visibility:hidden的区别

  • display: 隐藏对应的元素但不挤占该元素原来的空间，其子元素全部被隐藏，不能显现
  • visibility: 隐藏对应的元素并挤占该元素原来的空间，其子元素被隐藏后可设置visibility:visible来显示
